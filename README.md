# 🎯 Brand System - نظام الهوية الموحد

نظام متكامل لإدارة العلامات التجارية ومنصات التواصل الاجتماعي مع ميزات التخطيط السنوي وإدارة البودكاست.

## ✨ المميزات الرئيسية

### 📅 التخطيط السنوي (Yearly Planning)
- توزيع البرامج على 4 أرباع سنوية (Q1-Q4)
- سحب وإفلات البرامج بين الأرباع
- تصدير/استيراد JSON
- مولد قوالب الذكاء الاصطناعي
- إدخال جماعي للبرامج
- حفظ تلقائي في قاعدة البيانات

### 🎙️ إدارة البودكاست (Podcast Management)
- إضافة حلقات مع رفع ملفات صوتية
- مشغل صوتي مدمج
- ترقيم المواسم والحلقات
- تتبع الحالة (مسودة/مجدول/منشور)
- اكتشاف تلقائي لمدة الحلقة
- **الحلقات الافتراضية**:
  - زاوية المحلل - الحلقة الأولى
  - تحليل السوق للربع الأول
  - مستقبل التقنية المالية

### 📱 إدارة منصات التواصل (Social Media)
- تفعيل/إلغاء تفعيل المنصات
- إضافة برامج 100 منشور
- بطاقات المنصات مع الإحصائيات
- تصدير/استيراد البيانات

### 📤 التصدير والاستيراد
- نسخ JSON للحافظة
- تحميل ملفات JSON
- رفع ملفات JSON
- لصق نصوص JSON
- إنشاء قوالب للذكاء الاصطناعي

## 🗄️ إعداد Supabase

### الخطوة 1: إنشاء مشروع

1. اذهب إلى [https://supabase.com](https://supabase.com)
2. انقر **"Start your project"** أو **"New Project"**
3. املأ المعلومات:
   - **الاسم**: `brand-system`
   - **كلمة المرور**: اختر كلمة مرور قوية
   - **المنطقة**: اختر الأقرب لموقعك
4. انقر **"Create new project"**

### الخطوة 2: الحصول على مفاتيح API

1. في لوحة Supabase، انقر **Settings** (أيقونة الترس)
2. انقر **API** في القائمة الجانبية
3. انسخ القيم:
   - **Project URL** (مثال: `https://xxxxx.supabase.co`)
   - **anon/public key** (سلسلة طويلة تبدأ بـ `eyJ...`)

### الخطوة 3: تكوين البيئة

افتح ملف `.env` والصق معلوماتك:

```env
VITE_SUPABASE_URL=https://your-project-id.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...your-key
```

## 📊 تكوين قاعدة البيانات

### الخطوة 1: فتح محرر SQL

1. في لوحة Supabase، انقر **SQL Editor** (أيقونة </>)
2. انقر **"New query"**

### الخطوة 2: تشغيل الترحيل

1. افتح ملف `supabase-schema.sql` من مجلد المشروع
2. انسخ كل كود SQL
3. الصق في محرر SQL في Supabase
4. انقر **"Run"** أو اضغط `Ctrl+Enter`
5. انتظر رسالة النجاح

### الخطوة 3: التحقق من الإعداد

شغل هذا الاستعلام للتحقق من الجداول:

```sql
SELECT table_name 
FROM information_schema.tables 
WHERE table_schema = 'public'
ORDER BY table_name;
```

يجب أن ترى:
- `platforms`
- `programs`
- `podcast_episodes`
- `social_posts`

### الخطوة 4: إنشاء مساحة تخزين للملفات الصوتية

1. اذهب إلى **Storage** في لوحة Supabase
2. انقر **"New bucket"**
3. الاسم: `podcast-audio`
4. فعل **"Public bucket"** = ON
5. انقر **"Create bucket"**

## 🔐 ملاحظات الأمان

### الإعداد الحالي (للتطوير)
- سياسات RLS تسمح بالقراءة/الكتابة العامة
- أي شخص لديه مفتاح anon يمكنه تعديل البيانات
- **غير مناسب للإنتاج**

### للإنتاج
1. فعّل المصادقة (Supabase Auth)
2. حدّث سياسات RLS لتقييد الوصول حسب المستخدم
3. لا تلتزم ملف `.env` في Git
4. استخدم متغيرات البيئة في المنصة
