# 🔐 نظام التحكم في الظهور - Visibility Control System

## نظرة عامة

نظام متكامل يسمح للأدمن بالتحكم في:
- ✅ إظهار/إخفاء الأقسام للزوار
- ✅ تحديد ما يتطلب تسجيل دخول
- ✅ التحكم في كل ميزة بشكل منفصل
- ✅ معاينة التغييرات فوراً

---

## 📋 المميزات

### 1. **لوحة تحكم للأدمن**
- صفحة `/admin/visibility`
- واجهة سهلة بتبديلات (toggles)
- فلترة حسب الحالة
- بحث عن الأقسام

### 2. **أنواع التحكم**

#### أ. الظهور (Visibility)
- **ظاهر (Visible)** - يظهر للجميع
- **مخفي (Hidden)** - لا يظهر لأحد

#### ب. المصادقة (Authentication)
- **عام (Public)** - لا يتطلب دخول
- **خاص (Private)** - يتطلب تسجيل دخول

### 3. **الأقسام القابلة للتحكم**

#### المنصات الرئيسية
- منصة المستثمر الإخبارية
- أكاديمية المستثمر
- رادار المستثمر
- مساعد الإطلاق
- المستثمر سعودية

#### الميزات
- التخطيط السنوي
- البودكاست
- منصات التواصل
- التصدير/الاستيراد
- قوالب الذكاء الاصطناعي

3. الصق واضغط **"Run"**

```sql
SELECT * FROM visibility_settings ORDER BY display_order;
```

يجب أن ترى 25+ قسم قابل للتحكم.

### الخطوة 3: تسجيل دخول كأدمن

1. اذهب إلى `/login`
2. سجل دخول بحساب الأدمن
3. اذهب إلى `/admin/visibility`

## 📖 كيفية الاستخدام

### التبديلات (Toggles)

كل قسم له تبديلان:

#### 1. تبديل الظهور (الأخضر)
- **مفعل** ✅ - القسم ظاهر
- **معطل** ❌ - القسم مخفي

#### 2. تبديل المصادقة (الأصفر)
- **مفعل** 🔒 - يتطلب تسجيل دخول
- **معطل** 🔓 - متاح للجميع

### أمثلة استخدام

#### مثال 1: إخفاء منصة كاملة
```
1. ابحث عن "منصة المستثمر الإخبارية"
2. عطّل تبديل الظهور (الأخضر)
3. النتيجة: المنصة لن تظهر لأحد
```

#### مثال 2: جعل ميزة للأعضاء فقط
```
1. ابحث عن "التخطيط السنوي"
2. فعّل تبديل الظهور (الأخضر)
3. فعّل تبديل المصادقة (الأصفر)
4. النتيجة: تظهر فقط للمستخدمين المسجلين
```

#### مثال 3: إخفاء كل شيء ما عدا المنصات
```
1. اضغط "إخفاء الكل"
2. فعّل المنصات واحد واحد
3. النتيجة: موقع بسيط بمنصات فقط
```

## 🎯 سيناريوهات الاستخدام

### سيناريو 1: موقع عام (Public Site)
**للجميع بدون تسجيل دخول**

```
✓ جميع المنصات: ظاهر، لا يتطلب دخول
✓ البودكاست: ظاهر، لا يتطلب دخول
✗ التخطيط: مخفي
✗ التواصل: مخفي
✗ التصدير: مخفي
```

### سيناريو 2: موقع للأعضاء (Members Only)
**يتطلب تسجيل دخول**

```
✓ جميع الأقسام: ظاهر، يتطلب دخول
✗ التسجيل معطل
✓ الأدمن يضيف مستخدمين يدوياً
```

### سيناريو 3: هجين (Freemium)
**بعضه مجاني وبعضه مميز**

```
✓ المنصات: ظاهر، لا يتطلب دخول
✓ البودكاست: ظاهر، لا يتطلب دخول
✓ التخطيط Q1: ظاهر، لا يتطلب دخول
✗ التخطيط Q2-Q4: ظاهر، يتطلب دخول
✗ التواصل: ظاهر، يتطلب دخول
✗ التصدير: ظاهر، يتطلب دخول
```

## 🔧 البرمجة

### التحقق من الظهور في الكود

```typescript
// في أي مكون
import { supabase } from '../lib/supabase';

const { data } = await supabase
    .from('visibility_settings')
    .select('is_visible, requires_auth')
    .eq('section_key', 'feature_planning')
    .single();

if (!data?.is_visible) {
    // القسم مخفي، لا تعرضه
    return null;
}

if (data?.requires_auth && !user) {
    // يتطلب دخول والمستخدم غير مسجل
    return <LoginPrompt />;
}

// اعرض القسم
return <Section />;
```

### استخدام الدوال المساعدة

```sql
-- هل القسم ظاهر؟
SELECT is_section_visible('feature_planning');
-- النتيجة: true/false

-- هل المستخدم لديه وصول؟
SELECT has_section_access('feature_planning');
-- النتيجة: true/false

-- احصل على الأقسام الظاهرة للمستخدم الحالي
SELECT * FROM get_visible_sections();
```

### في React Components

```typescript
import { useAuth } from '../context/AuthContext';
import { supabase } from '../lib/supabase';

function MyComponent() {
    const { user } = useAuth();
    const [visible, setVisible] = useState(false);

    useEffect(() => {
        const checkVisibility = async () => {
            const { data } = await supabase
                .rpc('has_section_access', {
                    section_key_param: 'feature_planning'
                });
            
            setVisible(data);
        };

        checkVisibility();
    }, [user]);

    if (!visible) return null;

    return <div>محتوى القسم...</div>;
}
```

## 🔐 الصلاحيات

### من يمكنه التحكم؟

| الدور | عرض | تعديل |
|-------|-----|-------|
| زائر | ✅ | ❌ |
| مستخدم | ✅ | ❌ |
| أدمن | ✅ | ✅ |
| سوبر أدمن | ✅ | ✅ |

### حماية RLS

```sql
-- أي شخص يمكنه العرض
CREATE POLICY "Anyone can view" ON visibility_settings
    FOR SELECT USING (true);

-- الأدمن فقط يمكنه التعديل
CREATE POLICY "Admins can update" ON visibility_settings
    FOR UPDATE
    USING (
        EXISTS (
            SELECT 1 FROM user_profiles
            WHERE id = auth.uid() AND role IN ('admin', 'superadmin')
        )
    );
```

## 🐛 حل المشاكل

### "لا أستطيع رؤية صفحة الأدمن"

**الحل**: تأكد من أن دورك 'admin' أو 'superadmin'

```sql
SELECT role FROM user_profiles WHERE email = 'your@email.com';
```

### "التغييرات لا تظهر"

**الحل**: 
1. امسح الكاش
2. سجل خروج ودخول مجدداً
3. تحقق من RLS policies

### "أخطأ في SQL"

**الحل**: أعد تشغيل `supabase-visibility-schema.sql`
